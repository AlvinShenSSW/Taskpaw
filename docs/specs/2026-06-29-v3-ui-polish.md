# V3 UI/UX Polish — tracking spec (2026-06-29)

Tracking doc for the V3 frontend UI/UX polish pass. Visual baseline:
[`design-system/taskpaw-v3/preview/v3-ui-polish-preview.html`](../../design-system/taskpaw-v3/preview/v3-ui-polish-preview.html)
(dark full-app preview + 中/EN toggle + guided add-monitor wizard). Per-item
preview→code mapping:
[`v3-ui-polish-changelist.md`](../../design-system/taskpaw-v3/preview/v3-ui-polish-changelist.md).

Hard constraints (every issue): V2 frozen, V3-only under `taskpaw_v3/`; reuse the
existing design-system tokens (no new palette); **no emoji as icons** (SVG/MUI);
a11y (contrast ≥4.5:1, visible focus, status never color-only, reduced-motion
degrade, 150–300ms transitions); **Chinese is first-class** (CJK font fallback;
drop uppercase/wide letter-spacing under `lang=zh`); every behavior change ships a
test (`npm test` / `uv run pytest`); no PR while CI is red; no commit/push or
deploy unless the operator asks (CLAUDE.md).

## Suggested landing order

   StatusDot pulse, the Hub self-monitor tiles slice of #95, #92 skeletons.
2. **Theme base** (sets the global tone first): #89 tokens/fonts/grid + CJK.
3. **Wizard + form** (the big block — two PRs): #94 form templates first, then
   #93 the wizard shell. Adding a monitor (e.g. Lada) becomes the step-by-step
   guided flow from the preview.
4. **Hub depth**: #95 auto-refresh + acks-based health summary → drill-down;
   **#96 backend snapshot lands the per-machine metrics**.
