# V3 #17 — Plugin system + supervisor + process/heartbeat/tcp_check

Implements V3 design §4 (the Monitor plugin system) plus the three plugins the
moomoo preset (#18) needs.

## Modules (`taskpaw_v3/monitors/`)
- `base.py` — `MonitorPlugin` (type_id/category/config four-piece: pydantic
  model + json_schema + ui_schema + config_version + `create()`),
  `MonitorInstance` (`check(emit)→MonitorStatus`, `snapshot`, `reconfigure`),
  `EventEmitter` protocol, `BaseMonitorConfig` (resource caps: poll_interval,
  timeout, max_events_per_minute, max_line_bytes).
- `registry.py` — `PluginRegistry` + `default_registry()` (built-ins).
- `supervisor.py` — one worker thread per instance calling `check()` every
  `poll_interval`; `check()` failure → exponential backoff (5s..5min) + failure
  counter; 5 consecutive → DEGRADED + one alert; an `is_alive()` watchdog
  restarts dead worker threads; `emit` is per-instance throttled to
  `max_events_per_minute` (storm → one folded summary) and deduped by
  `dedupe_key`; `reconfigure` = stop→recreate→start; `snapshot()` reports
  state/alive/failures/degraded/dropped.

## Plugins (`taskpaw_v3/monitors/plugins/`)
- `process` — regex over psutil name/cmdline (covers plain processes, the PM2
  God Daemon `PM2.*God`, and the orchestrator script) without running `pm2`
  (which would spawn the daemon — #13). Emits alert/done on transitions.
- `heartbeat` — **status-aware** freshness (the #13 finding): `hibernating`
  status ⇒ not stale even if `next_check_due_utc` is days out; HUNG iff
  `now > next_check_due_utc + grace` AND not hibernating; falls back to file
  mtime + grace when no due field.
- `tcp_check` — connect-based port liveness (robust across OpenD's differing
  process names per platform).
