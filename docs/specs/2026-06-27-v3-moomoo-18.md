# V3 #18 — moomoo preset + end-to-end verification

Composes the #17 plugins into the four-life-signs preset (design §5.1) and wires
the supervisor into the agent. **Implementation + unit tests land here; the real
end-to-end verification runs on the moomoo machine (operator).**

## What's in this issue
- `taskpaw_v3/monitors/presets/moomoo.py` — `moomoo_preset(**overrides)` → the
  four monitor specs, with #13-confirmed defaults (all overridable).
- `taskpaw_v3/monitors/runtime.py` — `build_supervisor(registry, specs, queue,
  machine)` builds a Supervisor from `{type_id, config}` specs (pydantic-validated
  per plugin) and `make_queue_sink` bridges supervisor events into the agent's
  EventQueue (the same queue the Hub polls).
- Agent wiring — `run_agent` builds + starts the supervisor from
  `AgentConfig.monitors`, registers it for graceful shutdown, and reports
  `supervisor.snapshot()` in `/status`.

## The four life-signs (preset)
| # | sign | plugin | default |
|---|------|--------|---------|
| ① | pm2 God Daemon alive | `process` | pattern `PM2 .*God Daemon` |
| ② | orchestrator running | `process` | pattern `strategy_orchestrator\.py` |
| ③ | OpenD listening | `tcp_check` | `127.0.0.1:11111` |
| ④ | heartbeat fresh | `heartbeat` | `~/Documents/Workspace/moomoo/runtime/orchestrator_heartbeat.json`, status-aware, grace 300s |

## Deploy on the moomoo Mac (agent.yaml)
```yaml
server_id: moomoo-prod
machine: moomoo
bind_host: 192.168.x.x      # the LAN nic the Hub reaches (NOT 0.0.0.0)
bind_port: 5680
api_token: "<shared bearer>"
monitors:                   # paste moomoo_preset() output, or:
  - {type_id: process,   config: {name: moomoo-pm2-daemon, pattern: "PM2 .*God Daemon"}}
  - {type_id: process,   config: {name: moomoo-orchestrator, pattern: "strategy_orchestrator\\.py"}}
  - {type_id: tcp_check, config: {name: moomoo-opend, host: 127.0.0.1, port: 11111}}
  - {type_id: heartbeat, config: {name: moomoo-orchestrator-heartbeat, path: "~/Documents/Workspace/moomoo/runtime/orchestrator_heartbeat.json", grace_seconds: 300}}
```
Run headless: `python -m taskpaw_v3.agent.server.service` (after installing the
`v3` extra). Add the server to the Hub with this LAN IP + token.

## End-to-end verification (operator — needs the live moomoo box + OpenClaw)
Read-only agent; these manipulate moomoo, so do them on a quiet window / distill copy.

1. **OpenD down (③):** quit `moomoo_OpenD.app` (frees `:11111`) → expect a
   `moomoo-opend down` alert reach OpenClaw within ~1 poll; relaunch → `done`.
2. **orchestrator stopped (②):** `pm2 stop orchestrator` → `moomoo-orchestrator
   down` alert; `pm2 start orchestrator` → recovery.
3. **pm2 daemon (①):** `pm2 kill` (stops the God Daemon) → `moomoo-pm2-daemon
   down`. (This also stops everything pm2-managed — do last.) `pm2 resurrect`.
4. **heartbeat HUNG (④):** with the orchestrator NOT hibernating, let the
   heartbeat go past `next_check_due_utc + grace` (or temporarily point the
   monitor at a stale copy) → `moomoo-orchestrator-heartbeat unhealthy` alert.
   Confirm a `hibernating` status does NOT alert even with a far-future due time.

Each should produce one alert at the Hub → OpenClaw (and a `done` on recovery).

## Notes
- moomoo already self-heals via `orch-watchdog`; TaskPaw only alerts (complementary).
- All four monitors use process/port/readable-heartbeat checks — no permissioned
  trading-layer files (#13).
