## What's in this issue
- `taskpaw_v3/ui/` тАФ Vite + React 19 + MUI 6 frontend:
  - `theme.ts` from `design-system/taskpaw-v3/MASTER.md` (dark OLED, navy slate
    + blue accent, Fira Code/Sans) + status colors from the agent-console page.
  - `AgentConsole` (per `pages/agent-console.md`): two-pane тАФ left rail of this
    machine's monitors with status dots, main pane = selected monitor's live
    status + metrics. Polls the agent `/status`.
  - `HubDashboard` (per `pages/hub-dashboard.md`): fleet grid of machines + the
    Hub's own host-health self-monitor. Polls the Hub `/status`.
  - `SchemaForm` (RJSF + MUI): schema-driven monitor config (design ┬з4.3), fed by
    each plugin's `json_schema`.
  - `api.ts`: REST client; base URL + Bearer injected on the loopback origin in
    the shell, or via Vite env in the browser.
- `taskpaw_v3/src-tauri/` тАФ Tauri v2 shell:
  - `tauri.conf.json`: `withGlobalTauri=false`, CSP restricted to the loopback
    backend + the Google-Fonts origin; bundles `../ui/dist`.
  - `capabilities/default.json`: **empty permissions** тАФ no IPC/FS/shell exposed
    to the webview (┬з3.1). The UI is pure web тЖТ local backend over HTTP.
  - `src/main.rs`: spawns the backend child (`TASKPAW_BACKEND_CMD`) and **kills it
    on `ExitRequested`** (closing the last window). This is "X = exit" with no
    orphan process / held port (┬з7.1) тАФ the V2 tray-zombie problem, gone.

## Operator handoff (visual + native build)
1. Generate icons: `cargo tauri icon <a 1024px png>` (populates `src-tauri/icons/`).
2. Install the Tauri CLI: `npm i -g @tauri-apps/cli` (or `cargo install tauri-cli`).
3. Dev run: `cargo tauri dev` (from `taskpaw_v3/src-tauri/`). Set
   `TASKPAW_BACKEND_CMD` to launch the agent/hub backend, or run it separately.
4. Build the app: `cargo tauri build`.
5. **Visually verify** the agent console + Hub dashboard against the design system
   (dark contrast, status colors, no emoji icons, responsive), and confirm
   **X closes the window тЖТ the backend child exits** (no leftover process/port).
